Hyper Text Markup Language

  • 웹표준

    • - 웹 사이트를 작성할 때 따라야 하는 공식 표준이나 기술 규격

  • 웹 접근성

    • 장애의 여부와 상관 없이 모두가 웹사이트를 이용할 수 있게 하는 방식

  • 크로스 브라우징

    • 모든 브라우저 또는 기기에서 사이트가 제대로 작동하도록 하는 기법

  • HTML 주석은 ( < !-- 내용 --> ).

  • 웹사이트에서 눈에 보이는 정보나 특정 구역을 설정할 때 사용하는 언어.

HTML 문서의 기본 구조

  • head 태그
    1. 웹사이트의 간단한 요약 정보를 담는 영역
    2. 웹사이트에서 노출되지 않는 정보
  • body 태그
    1. 웹사이트에서 눈에 보이는 정보를 담는 영역
    2. 이미지나 텍스트처럼 출력되는 정보
  • meta 태그
    1. character setting의 약자를 나타내는 문자코드
    2. 모든 문자를 웹 브라우저에서 깨짐 없이 표시하겠다는 의미
  • title 태그
    1. 웹사이트 탭에 나타나는 제목을 적는 태그
  • img 태그
    1. img 태그 - 정보성을 갖고 있는 이미지를 삽입. 닫힌 태그 없음.
    2. src 속성 - 삽입할 이미지 파일 경로.
    3. alt 속성 - (웹표준. 필수!!) 웹사이트가 이미지를 출력하지 못했을 경우, 텍스트 정보로 대체.
  • h태그
    1. Heading의 약자로 제목이나 부제목을 표현
    2. 숫자 값이 클수록 폰트 사이즈가 작음. 즉, 숫자는 정보의 중요도를 나타냄.
    3. h1태그는 가장 중요한 정보를 담으므로, 하나의 html 문서에서 한 번만 사용됨.
  • p 태그
    1. Paragraph의 약자로 본문 내용을 표현
    2. 웹사이트의 중요 정보를 담는 태그
    3. 나타내고자 하는 내용을 열린 태그와 닫힌 태그 사이에 입력
  • ul 태그
    1. Unordered list의 약자로, 순서가 없는 리스트 생성.
    2. 메뉴 버튼을 만들 떄 사용 되는 태그

구조를 잡을 때 사용하는 태그

  • (목차)header > nav 태그
    1. header 태그

      2. - 웹사이트의 머리글을 담는 공간

    2. nav태그

      3. - 메뉴 버튼을 담는 공간 (navigation)

      - ul, li, a 태그와 함께 사용

  • (본문)main > article 태그
    1. main 태그

      2. - 문서의 주요 내용을 담는 태그

      - role="main" 속성 필수 입력!!

    2. article 태그

      3. - 문서의 주요 이미지나 텍스트 등의 정보를 담고 구역을 설정하는 태그

      - 태그 내에 구역을 대표하는 타이틀 < h#>태그가 존재해야함

    3. footer 태그

      4. - 가장 하단에 들어가는 정보를 표기할 때 사용

    4. div 태그

      5. - 임의의 공간을 만들 때 사용

Block 요소와 Inline 요소

두 요소를 구분 짓는 세 가지 주요 특징

  1. 줄바꿈 현상
  2. 가로|세로 배치
  3. 상|하 배치
  1. Block 요소

    - y축 정렬 형태로 출력 (줄바꿈 현상 나타남)

    - 공간을 만들 수 있고, 상하 배치 작업 가능

    - width, height 설정 가능

  2. Inline 요소

    x축 정렬 형태로 출력 (한 줄에 출력)

    공간을 만들 수 없고, 상하 배치 작업 불가능
